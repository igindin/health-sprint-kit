Твоя система здоровья — старт за 15 минут

Ты управляешь всем через Claude Code или Codex. Говоришь — он логирует твои данные, находит паттерны, помогает собрать твою систему. Учить интерфейс не нужно. Данные остаются на твоей машине.

Старт может быть простым: айфон и один вопрос про себя. А любишь глубину и большие массивы данных - велком, это тоже твой путь.

Если у тебя ещё нет своей Health OS — делай так

Это главный маршрут для новичка. Четыре шага, ничего ставить не надо. Начни с одного: скопируй промпт ниже своему агенту.

Шаг 1 · Скопируй это своему агенту
Илья дал ссылку на health-sprint-kit: https://github.com/igindin/health-sprint-kit. Я новичок, своей Health OS ещё нет. Не ставь всё подряд. Сначала прочитай README и skill openhealth-starter. Помоги мне собрать минимальную локальную health-папку: цель, контекст, источники данных, один паттерн и простой ежедневный check-in. Не давай медицинских рекомендаций и не копируй чужие skills без адаптации.
  1. Отдай промпт агенту
    Открой свою папку в Claude Code (или Codex), вставь промпт выше. Агент прочитает README и skill openhealth-starter и поведёт тебя по одному вопросу за раз.
  2. Пройди openhealth-starter
    Это один skill, который собирает первую систему за проход: твоя цель, контекст про тебя, какие данные уже есть. Без движка, только файлы.
  3. Выбери один паттерн и один вопрос
    «Что ломает мой сон?» / «Почему по средам нет сил?». Один, не три. Агент поможет поставить маленький n-of-1 эксперимент.
  4. Заведи простой ежедневный check-in
    Пара строк в день про самочувствие и паттерн. 20 секунд, без тяжёлого трекинга.

Результат первого дня — это папка, а не «установил движок». В ней лежат: goal.md, about-me.md, data-sources.md, checkin.md и AGENTS.md. Появились эти файлы — значит, ты сделал. Движок и импорт данных можно подключить позже.

Два пути — выбери свой

  • Лёгкий старт (рекомендуется): openhealth-starter, без движка, только файлы. Промпт агенту → один skill → один паттерн → ежедневный check-in. Этого достаточно, чтобы начать. Большинству хватает надолго.
  • Технический путь: OpenHealth repo + make setup + импорт Apple Health (позже, по желанию). Когда захочешь автоматическое хранилище и парсеры: клонируешь движок OpenHealth, один раз make setup, дальше импорт выгрузки Apple Health. Папка, собранная на лёгком старте, ляжет в него без переделок.

Технический путь — подробнее

Это для второго пути, когда захочешь движок и автоматический импорт. Новичку сюда идти не обязательно — лёгкого старта выше достаточно.

  1. Клонируй движок и запусти setup
    Склонируй OpenHealth repo, один раз выполни make setup — поставит движок и создаст локальное хранилище.
  2. Импортируй Apple Health (по желанию)
    iPhone → Здоровье → твоё фото → «Экспортировать данные» → .zip. Дальше движок разложит выгрузку по дням и слоям. Можно начать и с одного анализа.
  3. Перенеси папку с лёгкого старта
    Твои goal.md / about-me.md / checkin.md ложатся в движок без переделок — онтология и неминг те же.

  • openhealth-starterначни отсюда — первая система за один проход

Промты на потом — когда система уже стоит

Первый промпт для старта — наверху страницы. Эти — когда у тебя уже есть папка и захочется копнуть. Копируй и подставляй своё.

Всё разбросано → собрать в одно
Мои данные о здоровье лежат в разных местах: таблицы, заметки в чатах, папки, календарь, экспорт телефона. Собери их в один таймлайн, разложи по слоям и покажи, где что и чего не хватает.
Импорт Apple Health
У меня есть экспорт Apple Health в ~/Downloads (.zip). Импортируй его, разложи по дням и слоям, покажи сводку: какие метрики и за какой период.
Скачет энергия / по средам нет сил
У меня скачет энергия, и не всегда понятно от чего (не только сон или цикл). Собери n-of-1 на 2 недели: свяжи самочувствие с календарём, нагрузкой и сном — помоги найти, от чего. Как гипотезы, не как факт.
Сон: что его улучшает
Хочу отследить сон и понять, что его реально улучшает, а что ломает. Поставь простой трекинг и один эксперимент на неделю с ежедневным чек-ином.
Безопасность нового протокола
Я начал новый протокол по здоровью (препарат/добавка/режим). Какие маркеры безопасности отслеживать и как часто? Сведи в чеклист с источниками, уровнями доверия C1–C5 и красными флагами. Без диагнозов; высокий риск — к врачу.
Носимое + анализы → корреляции
У меня есть данные с носимого (HRV, пульс покоя, сон) и периодические анализы крови. Найди повторяющиеся связи между сном / HRV / нагрузкой и самочувствием. Только как осторожные гипотезы, с уровнем доверия.

Это шаблоны «под себя» — подставь своё. Готовые промпты intake (анализы из почты, скриншоты вместо интеграций) — внутри скилов health-journal и lab-interpretation.

Три правила

  • Помощник для размышления, не врач. Без диагнозов. Решения — с врачом. Тревожное → стоп и к специалисту.
  • У каждого вывода есть уровень доверия (C1–C5). Низкий — формулируется как вопрос, а не факт.
  • Не копируй чужой скил вслепую. Репозиторий может быть сложнее, чем нужно, или под другую цель. Адаптируй под свою.
